# {{name}}

## 目录结构

```
./
├── bin/                    # 存放编译后的可执行文件
├── build/                  # 编译生成的中间文件 (.o, .dep)
├── docs/                   # 项目文档
├── include/                # 存放.h或.hpp头文件
│   └── {{name}}/
│       └── {{name}}.h
├── res/                    # 存放图片或其他资源
├── src/                    # 存放.cpp源文件
│   └── {{name}}.cpp
├── tests/                  # 测试代码
├── tools/                  # 项目支持工具目录
├── lib/                    # 存放第三方库文件
├── CMakeLists.txt          # 构建配置文件
├── conanfile.py            # Canon配置文件
├── LICENSE                 # 项目许可证文件
└── README.md               # 项目说明
```

详细目录说明：

- `src/`: 存放所有的源代码文件 (`.cpp`)，通常按模块划分子目录。
- `include/`: 存放头文件 (`.h`, `.hpp`)。为了方便维护，通常会在 include 目录下再建立一个与项目同名的子目录。
- `build/`: 专用于存放CMake或Make产生的临时中间文件，保证源码目录的整洁。不同的构建配置（如 Debug 和 Release）会有各自的子目录。
- `bin/`: 存放最终生成的二进制可执行文件。也可用于存放构建与部署相关脚本。
- `tests/`: 存放单元测试或集成测试代码。
- `lib/`: 存放外部链接库文件。
